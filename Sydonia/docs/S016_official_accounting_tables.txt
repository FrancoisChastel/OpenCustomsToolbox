Introduction

The following document presents the technical description of the database tables that contain financial information (payments,
accounts, receipts).


Stucture of the document

This document gives a detailed description of each database table Notes regarding following table data: The data is presented in the
following format.

Field Name - DOM Representation - Null? - Data Type - Description

The Field Name is the name given to the data element in the table.

The DOM Representation is the Data set structure associated with the Field.

Null? shows whether the Field Value could be empty or not.

Data Type indicates the type and length of data being stored, and how the engine is storing it.

This is useful as an indication of the type validation performed against each element.

The Length is to be treated with caution, where data is stored in a numeric format, the length indicated is that for the table column.
It may seem different from the length available in the interface screens, but this is because the engine formats the data so that it is
stored in its most efficient manner. In particular, the date length seems meaningless. This simply indicates that the data is stored in the
database server date format, which incorporates the necessary structure and length.

Description is a brief explanation of the Field.

The table columns are presented in the same order that they appear in the actual database tables.
